Architecture Decision Records
핵심 설계 결정 3가지
ADR-001
국립중앙도서관 API
+ Mock Fallback
API 실패 시 내장 Mock 데이터로 자동 전환. 추가 비용 없이 수십만 권 메타데이터를 활용. CORS 이슈로 웹 환경에서는 Mock이 기본으로 동작.
▶ 비용 절감 + 안정성 확보
ADR-002
Google Gemini API
3단계 폴백 체인
2.5 Flash → flash-latest → 2.0-lite 순 자동 전환. 할당량 초과·서비스 장애 시에도 앱이 멈추지 않음. API 키 미설정 시 Mock 응답으로 대체.
▶ 무중단 AI 추천
ADR-003
AsyncStorage
로컬 퍼시스턴스
서버 없이 MVP를 빠르게 개발하기 위해 선택. 최근 본 책(20권), 구매 이력, 캐시, 카드 정보 로컬 저장. 키는 riv_ 접두사로 통일.
▶ 서버 의존도 최소화
